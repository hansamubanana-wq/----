Python + ADB + OpenCV

プロスピA 自動化ボット

ADB を使ってAndroidデバイスを操作し、プロ野球スピリッツAのゲームを自動で周回するシステム

概要 アーキテクチャ ファイル構成 動作モード テンプレート セットアップ 処理フロー

📋システム概要

このシステムはPC上のPythonスクリプトが ADB (Android Debug Bridge) 経由でスマートフォンを遠隔操作し、プロスピAのゲーム画面をスクリーンショット→画像認識→タップ という流れで自動周回するボットです。

対象ゲーム

プロ野球スピリッツA（Android版）。試練モードとVロードモードの2種類を自動化。

動作原理

ADBでスクショを取得 → OpenCVでUI部品を画像認識 → ADBでタップ送信。この繰り返し。

接続方式

USBケーブル接続またはWi-Fi（ワイヤレスデバッグ）の両方に対応。

主な自動処理

難易度選択・試合開始・結果スキップ・スタミナ回復・ラウンド結果処理など。

📁ファイル構成

config.py 設定ファイル ADBパス・デバイスシリアル・解像度・タイミング・マッチング閾値などの定数

🎮動作モード

起動時に1または2を入力してモードを選択。どちらも Ctrl+C で停止するまで無限ループ。

① 試練モード

初級→中級→上級 を順番にローテーションして自動周回する。

  • 難易度ボタン選択（初級/中級/上級）
  • 「自動試合開始」をタップ
  • 試合終了後の結果画面スキップ
  • スタミナ不足時に80回復
  • 難易度選択画面に戻るまで待機

② Vロードモード

Vロード専用の結果画面・ラウンド結果を処理しながら周回。

  • 「自動試合開始」をタップ
  • ラウンド結果画面を3回タップで消去
  • 「次の試合へ」ボタンを検出してタップ
  • Vロード専用「次へ」ボタンを処理
  • 5秒間無反応時にアイドルタップ
Vロードモードは btn_next_match・btn_next_vroad・screen_round_result の3つのテンプレートが必須。未登録の場合は起動時にエラーメッセージで停止する。

マッチングアルゴリズム

OpenCVの cv2.TM_CCOEFF_NORMED を使用。グレースケール変換後に正規化相互相関でスコアを算出し、閾値（デフォルト 0.80）以上なら一致と判定する。

# screen.py より抜粋
screen_gray = cv2.cvtColor(screen_img, cv2.COLOR_BGR2GRAY)
template_gray = cv2.cvtColor(template, cv2.COLOR_BGR2GRAY)
result = cv2.matchTemplate(screen_gray, template_gray, cv2.TM_CCOEFF_NORMED)
_, max_val, _, max_loc = cv2.minMaxLoc(result)
if max_val >= threshold:
    center_x = max_loc[0] + w // 2
    center_y = max_loc[1] + h // 2
    return (center_x, center_y)

⚙️セットアップ

1. 依存ライブラリのインストール

setup.bat  # ダブルクリックで実行
# または
pip install opencv-python pillow numpy

2. config.py の設定

ADB_PATH = r"C:\...\adb.exe"      # ADBの場所を指定
DEVICE_SERIAL = None               # 1台接続ならNoneでOK
SCREEN_WIDTH = 1080                # デバイスの解像度
SCREEN_HEIGHT = 2400

3. デバイス接続確認

python check_device.py

4. テンプレート画像の登録

python capture_tool.py
# スクショが表示されるのでドラッグで範囲選択 → S で保存

5. ボット起動

python bot.py
# 1: 試練  2: Vロード  を選択
Wi-Fi接続の場合： スマホの「ワイヤレスデバッグ」を有効にして adb connect <IP>:5555 を実行後、config.py の DEVICE_SERIAL にIPアドレスを設定する。

🔄処理フロー詳細

試練モード

1
難易度ボタン選択

初級→中級→上級 を順番にローテーション。ボタンが見つからなければ手動入力待ち

2
自動試合開始

btn_auto_start を最大15秒待機してタップ

3
試合終了〜結果処理（最大5分）

スタミナ不足検出→回復、TAP SCREEN→タップ、次へボタン→タップ を繰り返す

4
難易度選択画面に戻ったら1へ戻る

いずれかの難易度ボタンが検出されれば1周完了

Vロードモード

1
自動試合開始

btn_auto_start をタップ

2
試合中〜ラウンド結果処理（最大5分）

ラウンド結果画面では1秒間隔で3回タップ。次の試合へ / Vロード次へ / 通常次へ を優先順位付きで検出

3
アイドルタップ機構

5秒間何も検出されない場合、画面上部（300px）を安全タップしてUIをリセット

4
btn_auto_start が戻ってきたら1へ

自動試合開始画面を検出したら1周完了
